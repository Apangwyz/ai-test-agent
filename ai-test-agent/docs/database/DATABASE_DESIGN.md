# 数据库设计文档

## 1. 数据库概述

本项目使用 **PostgreSQL** 作为主数据库，主要用于存储用户认证信息。同时，项目还使用文件系统存储知识库数据，使用 FAISS 作为向量数据库存储嵌入向量。

## 2. 数据库表结构

### 2.1 users 表

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| `id` | `Integer` | `PRIMARY KEY, INDEX` | 用户ID |
| `username` | `String(50)` | `NOT NULL` | 用户名 |
| `email` | `String(100)` | `UNIQUE, NOT NULL, INDEX` | 邮箱地址 |
| `password_hash` | `String(255)` | `NOT NULL` | 密码哈希值 |
| `role` | `String(20)` | `NOT NULL, DEFAULT 'user'` | 用户角色 |
| `created_at` | `DateTime(timezone=True)` | `DEFAULT NOW()` | 创建时间 |
| `updated_at` | `DateTime(timezone=True)` | `ON UPDATE NOW()` | 更新时间 |

### 2.2 表关系图

```
+------------------+
|      users       |
+------------------+
| id (PK)          |
| username         |
| email (UNIQUE)   |
| password_hash    |
| role             |
| created_at       |
| updated_at       |
+------------------+
```

## 3. 数据类型说明

| 数据类型 | 描述 | 用途 |
|---------|------|------|
| `Integer` | 整型 | 用于ID等数值字段 |
| `String(n)` | 字符串，最大长度n | 用于存储用户名、邮箱、密码哈希等 |
| `DateTime(timezone=True)` | 带时区的日期时间 | 用于记录创建和更新时间 |

## 4. 索引设计

| 表名 | 索引字段 | 索引类型 | 用途 |
|------|---------|---------|------|
| `users` | `id` | 主键索引 | 加速用户ID查询 |
| `users` | `email` | 唯一索引 | 加速邮箱查询，确保唯一性 |

## 5. 数据存储方案

### 5.1 关系型数据库
- **PostgreSQL**：存储用户认证信息
- **连接方式**：通过 SQLAlchemy ORM 连接
- **连接字符串**：`postgresql://admin:password@localhost:5432/example_db`

### 5.2 文件系统存储
- **知识库数据**：存储在 `src/knowledge_base/data/` 目录
- **存储格式**：JSON 文件
- **结构**：按知识类型分类存储

### 5.3 向量数据库
- **FAISS**：用于存储和检索文本嵌入向量
- **存储位置**：内存 + 磁盘文件
- **索引类型**：FlatL2（适合小规模数据）
