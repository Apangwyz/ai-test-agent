系统概述

AI智能体系统是一个功能强大的智能助手平台，旨在帮助用户处理文档、生成需求澄清、技术方案、编码任务和测试案例。系统采用现代化的前端界面和强大的后端处理能力，为用户提供一站式的AI辅助服务。

系统主要特点：

  • 支持多种文档格式的处理（DOCX、PDF、MD）
  • 智能生成需求澄清文档
  • 自动生成技术方案文档
  • 基于技术方案生成编码任务
  • 根据需求和技术方案生成测试案例脑图
  • 现代化的用户界面，支持响应式设计
  • 安全的用户认证系统

核心功能

1. 文档处理

上传并处理各种格式的文档，系统会自动提取结构化数据，为后续的流程提供基础。

  • 支持DOCX、PDF、MD格式
  • 自动提取关键信息
  • 生成结构化数据

2. 需求澄清

基于结构化的需求数据，自动生成详细的需求澄清文档，帮助用户明确需求细节。

  • 智能识别模糊需求
  • 生成标准化的澄清文档
  • 提供需求分析建议

3. 技术方案生成

根据结构化需求和澄清文档，自动生成完整的技术方案文档，包括架构设计、技术选型等。

  • 基于需求自动设计技术架构
  • 推荐合适的技术栈
  • 生成详细的实现方案

4. 编码任务生成

根据技术方案文档，自动分解和生成详细的编码任务，为开发团队提供明确的工作指南。

  • 任务分解与优先级排序
  • 详细的任务描述和要求
  • 预估工作量和时间

5. 测试案例生成

基于需求和技术方案，自动生成测试案例脑图，帮助测试团队全面覆盖测试场景。

  • 生成完整的测试用例
  • 覆盖功能和非功能测试
  • 提供测试优先级建议

快速开始

步骤1：环境准备

确保您的系统满足以下要求：

  • Python 3.10+
  • pip 20.0+
  • 现代浏览器（Chrome、Firefox、Safari、Edge）

步骤2：安装依赖

在项目根目录执行以下命令：

pip install -r requirements.txt

步骤3：配置环境变量

复制 .env.example 文件为 .env，并填写必要的配置信息：

# 服务器配置 PORT=5002 # 数据库配置 DATABASE_URL=sqlite:///./test_weaver.db # 大模型配置 OPENAI_API_KEY=your_api_key MODEL_NAME=gpt-4

步骤4：启动服务器

执行以下命令启动服务器：

python app.py

服务器将在 http://localhost:5002 上运行。

步骤5：访问系统

打开浏览器，访问 http://localhost:5002，您将看到系统的登录页面。

使用指南

1. 用户认证

注册新账户

  1. 点击页面右上角的「登录」按钮
  2. 在登录模态框中点击「立即注册」链接
  3. 填写用户名、邮箱和密码
  4. 点击「注册」按钮
  5. 注册成功后，系统会自动跳转到登录页面

登录系统

  1. 点击页面右上角的「登录」按钮
  2. 填写邮箱和密码
  3. 点击「登录」按钮
  4. 登录成功后，您将看到系统的主界面

2. 文档处理

上传并处理文档

  1. 点击导航栏中的「文档处理」链接
  2. 点击「选择文件」按钮，选择要上传的文档（支持DOCX、PDF、MD格式）
  3. 点击「处理文档」按钮
  4. 等待系统处理完成，处理结果将显示在页面上
  5. 处理结果会自动填充到后续的表单中

3. 需求澄清

生成需求澄清文档

  1. 点击导航栏中的「需求澄清」链接
  2. 在「结构化需求数据」文本框中输入或粘贴结构化的需求数据
  3. 点击「生成澄清文档」按钮
  4. 等待系统生成完成，澄清文档将显示在页面上
  5. 生成的澄清文档会自动填充到技术方案表单中

4. 技术方案生成

生成技术方案文档

  1. 点击导航栏中的「技术方案」链接
  2. 在「结构化需求数据」文本框中输入或粘贴结构化的需求数据
  3. 在「澄清文档」文本框中输入或粘贴需求澄清文档（可选）
  4. 点击「生成技术方案」按钮
  5. 等待系统生成完成，技术方案文档将显示在页面上
  6. 生成的技术方案文档会自动填充到编码任务和测试案例表单中

5. 编码任务生成

生成编码任务文档

  1. 点击导航栏中的「编码任务」链接
  2. 在「技术方案文档」文本框中输入或粘贴技术方案文档
  3. 点击「生成编码任务」按钮
  4. 等待系统生成完成，编码任务文档将显示在页面上

6. 测试案例生成

生成测试案例脑图

  1. 点击导航栏中的「测试案例」链接
  2. 在「结构化需求数据」文本框中输入或粘贴结构化的需求数据
  3. 在「技术方案文档」文本框中输入或粘贴技术方案文档
  4. 点击「生成测试案例」按钮
  5. 等待系统生成完成，测试案例脑图将显示在页面上

常见问题

如何获取大模型API密钥？

您需要前往对应大模型提供商的官网注册账户并申请API密钥。例如，OpenAI的API密钥可以在OpenAI官网的个人设置中获取。获取后，将API密钥填写到 .env 文件中的 OPENAI_API_KEY 字段。

支持哪些文档格式？

系统目前支持以下文档格式：

  • DOCX（Microsoft Word文档）
  • PDF（ Portable Document Format）
  • MD（Markdown文档）
生成的文档如何保存？

系统会在页面上显示生成的文档内容，您可以直接复制粘贴到本地文件中保存。未来版本将支持直接下载生成的文档。

系统支持哪些大模型？

系统目前支持OpenAI的GPT系列模型，如GPT-3.5和GPT-4。您可以在 .env 文件中通过 MODEL_NAME 字段指定要使用的模型。

处理文档时出现错误怎么办？

如果处理文档时出现错误，请检查以下几点：

  • 文档格式是否支持（DOCX、PDF、MD）
  • 文档大小是否合理（建议不超过10MB）
  • 网络连接是否正常
  • 大模型API密钥是否有效

如果问题仍然存在，请查看服务器日志获取详细错误信息。

故障排除

常见错误及解决方案

1. 403 Forbidden错误

症状：访问系统时收到403 Forbidden错误。

解决方案：确保服务器配置正确，特别是静态文件的访问权限。检查 app.py 文件中的静态文件配置是否正确。

2. 端口被占用错误

症状：启动服务器时收到「Address already in use」错误。

解决方案：修改 .env 文件中的 PORT 字段，使用一个未被占用的端口。例如，将端口从5000改为5002。

3. API密钥错误

症状：生成文档时收到API密钥相关错误。

解决方案：检查 .env 文件中的 OPENAI_API_KEY 字段是否正确填写，确保API密钥有效且未过期。

4. 数据库连接错误

症状：启动服务器时收到数据库连接错误。

解决方案：检查 .env 文件中的 DATABASE_URL 字段是否正确配置。对于SQLite数据库，确保路径正确且具有写入权限。

5. 前端界面显示异常

症状：前端界面布局混乱或功能异常。

解决方案：清除浏览器缓存，或使用隐私模式打开页面。如果问题仍然存在，请检查浏览器控制台是否有JavaScript错误。
