# 需求文档5：银行网点智能柜面系统

## 项目名称
银行智慧网点柜面服务平台

## 项目背景
银行网点是服务客户的重要渠道，传统的柜面业务处理效率低、客户等待时间长。随着金融科技的发展，需要建设一套智能化的柜面系统，实现业务处理自动化、服务流程标准化、客户体验优质化，提升网点服务能力和运营效率。

## 核心功能需求

1. **个人业务处理**
   - 个人开户（借记卡、信用卡）
   - 存款、取款、转账
   - 密码修改、挂失、解挂
   - 账户信息变更
   - 个人结售汇业务

2. **对公业务处理**
   - 对公账户开户、变更、撤销
   - 对公转账、汇款
   - 支票业务（出票、承兑、贴现）
   - 银行承兑汇票业务
   - 对公账户信息查询

3. **客户身份识别**
   - 身份证读取与验证（OCR识别）
   - 人脸识别与比对
   - 联网核查（公安系统对接）
   - 客户画像显示
   - 风险等级提示

4. **业务预处理与分流**
   - 智能排队叫号系统
   - 业务预处理（填单、资料上传）
   - 客户识别与标签显示
   - 业务分流建议
   - 预约服务管理

5. **影像与凭证管理**
   - 业务凭证电子化
   - 现场拍照上传
   - 资料影像采集
   - 凭证要素自动识别
   - 影像质检与补扫

6. **柜员工作台**
   - 待处理任务列表
   - 常用交易快捷入口
   - 客户历史业务查看
   - 产品推荐展示
   - 柜员考勤与绩效

## 非功能需求

**性能要求：**
- 业务交易响应时间 < 1秒
- 人脸识别比对时间 < 500ms
- 身份证OCR识别时间 < 300ms
- 支持200+网点并发
- 凭证扫描速度 > 40页/分钟

**兼容性要求：**
- 支持Windows 10/11操作系统
- 支持主流柜面外设（高拍仪、指纹仪、密码键盘、IC卡读写器）
- 支持国密USB Key
- 支持柜面专用打印机

**安全性要求：**
- 柜员身份认证（指纹+密码+USB Key）
- 交易分级授权机制
- 重要交易双人复核
- 监控录像与业务关联
- 异常交易实时预警

## 目标用户群体
- 银行柜员
- 银行大堂经理
- 银行网点负责人
- 客户经理
- 运维支持人员
- 合规检查人员

## 技术栈要求
- 前端：WPF / Electron
- 后端：Java 17 + Spring Boot
- 数据库：Oracle / MySQL
- 缓存：Redis
- 消息队列：RabbitMQ
- 外设集成：Java Service Wrapper
- 影像处理：LibreOffice、PDF.js
- 生物识别：虹软、旷视SDK
- 打印驱动：POSPrinter SDK
